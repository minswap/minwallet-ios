# Contributing to MinWallet iOS

First off, thank you for considering contributing to MinWallet! It's people like you that make MinWallet such a great tool.

## Code of Conduct

This project and everyone participating in it is governed by the [MinWallet Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report for MinWallet. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Check the documentation** to see if you can find the solution yourself.
- **Search existing issues** to see if the bug has already been reported.
- **Collect information** about your environment (iOS version, device model, app version).
- **Provide a clear, step-by-step reproduction** of the issue.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion for MinWallet, including completely new features and minor improvements to existing functionality.

- **Search existing issues** to see if the enhancement has already been suggested.
- **Explain why this enhancement would be useful** to most MinWallet users.
- **Provide a detailed description** of the suggested enhancement.

### Your First Code Contribution

Unsure where to begin contributing to MinWallet? You can start by looking through these `good-first-issue` and `help-wanted` issues.

#### Local Development Setup

1. Clone the repository.
2. Follow the setup instructions in the [README.md](README.md).
3. Create a new branch for your changes.
4. Make your changes and ensure tests pass.
5. Submit a Pull Request.

## Swift Styleguide

We follow the [Swift API Design Guidelines](https://swift.org/documentation/api-design-guidelines/). Please ensure your code is formatted correctly by running:

```bash
swift format format -r -p -i MinWallet
```

## Commit Messages

- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.
- Reference issues and pull requests liberally after the first line.
